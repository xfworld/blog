## 背景 

本文档解读 Nathan Bossart 提交的 autovacuum 优先级排序改进，该功能根据表的需要程度自动计算优先级，确保最需要清理的表优先被 vacuum 处理。

| `771fe0948cad` | Avoid including vacuum.h in tableam.h and heapam.h |

### 核心价值

- **智能调度**：最紧急的表（如接近事务 ID 环绕的表）优先处理
- **可配置权重**：通过 GUC 调整各因素的权重
- **向后兼容**：将所有权重设为 0 可恢复原有行为

### 2. 评分算法

#### 2.1 `relation_needs_vacanalyze()` 函数 (line 3098-3365)

该函数计算表的需要程度分数，考虑以下因素：

| 组件 | 计算方式 | 公式 |
|------|----------|------|
| **XID 分数** | `xid_age / freeze_max_age` | 接近事务 ID 环绕风险 |
| **MXID 分数** | `mxid_age / multixact_freeze_max_age` | 接近多事务 ID 环绕风险 |
| **Vacuum 分数** | `dead_tuples / vac_threshold` | 死元组数量相对于阈值 |
| **Vacuum Insert 分数** | `ins_since_vacuum / vac_ins_threshold` | 自上次 vacuum 后的插入数 |
| **Analyze 分数** | `mod_since_analyze / analyze_threshold` | 自上次 analyze 后的修改量 |

#### 2.2 阈值计算 (line 3305-3311)

```c
/* Vacuum 阈值 */
vacthresh = vac_base_thresh + vac_scale_factor * reltuples
          = autovacuum_vacuum_threshold + autovacuum_vacuum_scale_factor * reltuples

/* Vacuum Insert 阈值 */
vacinsthresh = vac_ins_base_thresh + vac_ins_scale_factor * reltuples * pcnt_unfrozen
             = autovacuum_vacuum_insert_threshold + autovacuum_vacuum_insert_scale_factor * reltuples * pcnt_unfrozen

/* Analyze 阈值 */
anlthresh = anl_base_thresh + anl_scale_factor * reltuples
          = autovacuum_analyze_threshold + autovacuum_analyze_scale_factor * reltuples
```

#### 2.3 环绕风险激进缩放 (line 3258-3261)

当 XID/MXID 年龄超过安全阈值时，分数会被指数级放大：

```c
if (xid_age >= effective_xid_failsafe_age)
{
    /* 超过 10 亿行时，分数指数增长 */
    scores->xid = pow(scores->xid,
                      Max(1.0, (double) xid_age / 100000000));
}
```

#### 2.4 最终分数计算 (line 3266)

```c
/* 最终分数是所有组件的最大值 */
scores->max = Max(scores->xid, scores->mxid);
scores->max = Max(scores->max, scores->vac);
scores->max = Max(scores->max, scores->vac_ins);
scores->max = Max(scores->max, scores->anl);
```

### 3. 表收集与排序

#### 3.1 表添加到处理列表 (line 2092-2100, 2194-2201)

```c
if (dovacuum || doanalyze)
{
    TableToProcess *table = palloc_object(TableToProcess);
    table->oid = relid;
    table->score = scores.max;  /* 使用计算出的最高分数 */
    tables_to_process = lappend(tables_to_process, table);
}
```

#### 3.2 排序比较器 (line 1913-1920)

```c
static int
TableToProcessComparator(const ListCell *a, const ListCell *b)
{
    TableToProcess *t1 = (TableToProcess *) lfirst(a);
    TableToProcess *t2 = (TableToProcess *) lfirst(b);

    /* 降序排列 - 最高分的表优先处理 */
    return (t2->score < t1->score) ? -1 :
           (t2->score > t1->score) ?  1 : 0;
}
```

#### 3.3 排序触发 (line 2336)

```c
/* 在 worker 处理表之前排序 */
list_sort(tables_to_process, TableToProcessComparator);
```

### 4. 优先级决定因素示例

```
场景 1: 事务 ID 环绕风险
- 表 A: xid_age = 1,000,000, freeze_max_age = 200,000,000 → score = 0.005
- 表 B: xid_age = 2,000,000,000, freeze_max_age = 200,000,000 → score = 10 (指数放大后)

结果: 表 B 会被优先处理

场景 2: 死元组积压
- 表 C: 1000 死元组, vac_threshold = 50 → score = 20
- 表 D: 500,000 死元组, vac_threshold = 1000 → score = 500

结果: 表 C 会被优先处理

场景 3: 混合因素
- 表 E: 接近环绕 + 大量死元组
- 最终分数 = max(xid_score, mxid_score, vac_score, ...) = 最高组件
```

### 5. 恢复原有行为

如果所有权重设为 0，autovacuum 会恢复到之前的行为（按发现顺序处理）：

```c
/* line 2331-2336 */
if (autovacuum_freeze_score_weight == 0.0 &&
    autovacuum_multixact_freeze_score_weight == 0.0 &&
    autovacuum_vacuum_score_weight == 0.0 &&
    autovacuum_vacuum_insert_score_weight == 0.0 &&
    autovacuum_analyze_score_weight == 0.0)
{
    /* 跳过排序，保持原有顺序 */
}
else
{
    list_sort(tables_to_process, TableToProcessComparator);
}
```

## 使用实践

### 1. 查看表优先级分数

```sql
-- 查看接近环绕风险的表
SELECT
    relname,
    age(relfrozenxid) AS xid_age,
    mxid_age,
    n_dead_tup,
    n_live_tup,
    last_vacuum,
    last_autovacuum,
    vacuum,
    autovacuum
FROM pg_class
WHERE relkind = 'r'
ORDER BY age(relfrozenxid) DESC
LIMIT 20;
```

### 2. 配置权重参数

在 `postgresql.conf` 中调整各因素的权重：

```conf
# 默认值都是 1.0
autovacuum_freeze_score_weight = 1.0
autovacuum_multixact_freeze_score_weight = 1.0
autovacuum_vacuum_score_weight = 1.0
autovacuum_vacuum_insert_score_weight = 1.0
autovacuum_analyze_score_weight = 1.0

# 示例：更重视事务环绕风险
# autovacuum_freeze_score_weight = 2.0
# autovacuum_multixact_freeze_score_weight = 2.0

# 示例：更重视死元组清理
# autovacuum_vacuum_score_weight = 2.0
```

### 3. 监控 Autovacuum 活动

```sql
-- 查看当前正在进行的 vacuum 操作
SELECT
    pid,
    datname,
    relid::regclass,
    query,
    query_start,
    state
FROM pg_stat_activity
WHERE state != 'idle'
  AND query LIKE '%vacuum%';

-- 查看 autovacuum 工作进程
SELECT
    pid,
    datname,
    usename,
    application_name,
    state,
    query_start
FROM pg_stat_activity
WHERE application_name LIKE 'autovacuum worker%';
```

### 4. 强制关键表立即 Vacuum

对于紧急情况，可以手动触发 vacuum：

```sql
-- 对特定表进行紧急 vacuum
VACUUM FREEZE table_name;

-- 或者禁用该表的 autovacuum，手动管理
ALTER TABLE table_name SET (autovacuum_enabled = off);

-- 手动 vacuum
VACUUM ANALYZE table_name;

-- 重新启用 autovacuum
ALTER TABLE table_name SET (autovacuum_enabled = on);
```

## 总结

Autovacuum 优先级排序是期待已久的改进，它确保最需要处理的表（如接近事务 ID 环绕的表）会被优先 vacuum。该功能对 DBA 透明，自动生效，但可以通过权重参数进行细粒度控制。

**关键要点**：

1. **自动生效**：无需额外配置，默认行为已优化
2. **环绕保护**：接近事务 ID 环绕的表会被优先处理
3. **可调权重**：通过 GUC 调整各因素的优先级
4. **向后兼容**：所有权重设为 0 可恢复原有行为
5. **监控重要**：使用 `pg_stat_user_tables` 和 `pg_class` 监控表状态
