## 背景  
最近看AI行业论文趋势, 看到一篇较火的论文 "VibeVoice" : https://arxiv.org/abs/2508.19205  
  
仔细一看, 居然是语音生成和语音识别相关的, 而且模型也开源了, 生成播客配音的项目也开源了.  
  
正好拿来给我的文章生成播客, 最近豆包也不知道咋了, 不能把微信公众号文章直接丢过去生成播客了, 难道大厂又打起来了?  
  
话不多说, 这篇就手把手教大家玩转"Mac本地生成播客配音, 想要谁的声音都行, 还能带BGM"  
  
想关项目如下  
- https://github.com/microsoft/VibeVoice/blob/main/docs/vibevoice-tts.md  
- https://github.com/vibevoice-community/VibeVoice.git  
- https://github.com/digoal/VibeVoice    # 使用这个就好了, 我做了一些修改, 修正了一些问题.  
  
## 实操  
我的硬件: Mac Mini M2 16G  
  
1、确认环境  
```  
# 确认 Python 版本 >= 3.9  
python3 --version  
  
# brew  
brew  
```  
  
brew 参考  
- [《Claude 使用经验总结》](../202604/20260415_05.md)  
  
  
2、创建虚拟环境  
```  
cd ~  
python3 -m venv vibevoice_env  
  
# 进入虚拟环境  
source ~/vibevoice_env/bin/activate  
  
# 如何退出虚拟环境, 执行: deactivate

在虚拟环境中安装依赖.  
  
下次使用前, 请先进入以上虚拟环境  
```  
source ~/vibevoice_env/bin/activate  
```  
  
3、克隆项目  
```  
cd ~/

```  
  
  
4、安装依赖  
```

5、下载模型权重  
  
5\.1、 安装 ModelScope SDK  
  
```  
pip install modelscope  
```  
  
5\.2、 使用 Python 脚本下载模型  
  
在终端中进入 Python 交互环境（直接输入 python），然后逐行执行以下代码：  
  
下载 microsoft/VibeVoice-1.5B 模型权重到当前 `./models` 目录  
```  
from modelscope import snapshot_download  
  
# 下载 VibeVoice-1.5B 模型到本地 ./models/VibeVoice-1.5B 目录  
model_dir = snapshot_download('microsoft/VibeVoice-1.5B', cache_dir='./models')  
  
print(f"模型已下载到: {model_dir}")  
```  
  
下载 Qwen/Qwen2.5-1.5B 模型权重到当前 `./models` 目录  
```  
from modelscope import snapshot_download  
  
# 下载 Qwen2.5-1.5B 模型到本地 ./models/Qwen2.5-1.5B 目录  
model_dir = snapshot_download('Qwen/Qwen2.5-1.5B', cache_dir='./models')  
  
print(f"模型已下载到: {model_dir}")  
```  
  
执行后，脚本会显示下载进度，完成后会打印出模型的实际存储路径。  
  
5\.3、 确认模型文件  
  
下载完成后，可以在 `./models/microsoft/VibeVoice-1.5B` 目录下找到所有模型文件。  
  
注意, 位置可能是个软链.  
  
安装完后，准备播客文本文件（格式见之前说明），然后运行推理：  
  
使用前, 确认已进入虚拟环境.  
  
如果文本文件中有多个 speaker, 在 `--speaker_names` 参数中使用空格隔开即可.  
  
`demo/text_examples/1p_Ch2EN.txt` 只有一个 speaker, 输入多个声音文件不会报错, 只使用前面的那个声音文件

  --speaker_names zh-Bowen_man zh-Xinran_woman

7、如果想用自己的声音?  
  
找一段 10-30 秒的干净中文语音（无背景音乐、无噪音），保存为 `WAV` 格式，放到 `demo/voices/` 目录下，文件名就是 `speaker` 名字：  
```  
# 比如你用手机录了一段录音 digoal.m4a  
  
# 先转换成 wav 格式, 用 mac 自带的软件即可转换  
  
afconvert -f WAVE -d LEI16@24000 digoal.m4a digoal.wav  
```  
  
参数说明：  
```  
-f WAVE：输出 WAV 格式  
-d LEI16@24000：16-bit、24000 Hz 采样率（和 VibeVoice 一致，不需要重采样）  
```  
  
转完后直接放到 `demo/voices` 目录：  
```  
cp digoal.wav demo/voices/digoal.wav  
```  
  
然后直接用文件名（不带扩展名）作为 speaker_names  
```  
TOKENIZERS_PARALLELISM=true python demo/inference_from_file.py \
  --device mps \
  --model_path ./models/microsoft/VibeVoice-1.5B \
  --num_steps 6 \
  --txt_path demo/text_examples/1p_Ch2EN.txt \
  --speaker_names digoal  
```  
  
模型会自动提取你那段音频的音色来生成。  
  
补充：`Anchen` 带 `bgm` 是什么意思  
`zh-Anchen_man_bgm` 里的 `bgm` 表示这个参考音频本身带有背景音乐，用它生成的语音有概率也会带背景音乐，适合需要音乐氛围的场景，普通播客建议用 `zh-Bowen_man` 或 `zh-Xinran_woman`。

**问题1**：MPS 分支里 `torch_dtype=torch.float16` 但 `load_dtype` 定义时写的是 `float32`，存在矛盾，以哪个为准不确定。  
  
**问题2**：`model.set_ddpm_inference_steps(num_steps=10)` 硬编码，需要改成可配置。  
  
**问题3**：`attn_implementation` 重复传了两次（一个变量一个字面量）。  
  
直接给你改好的 patch，用 `str_replace` 精准替换：  
  
**第一处：修正 load_dtype 为 float16，并加 --num_steps 参数**文件在你机器上，我没有访问权限，直接告诉你改哪几行。对照你贴的代码，做以下 **3 处修改**：  
  
---  
  
### 修改 1：修正 MPS 的 dtype + 去掉重复的 `attn_implementation`  
  
找到这段（约第 195-205 行）：  
  
```python  
    # Decide dtype & attention implementation  
    if args.device == "mps":  
        load_dtype = torch.float32  # MPS requires float32  
        attn_impl_primary = "sdpa"  # flash_attention_2 not supported on MPS  
```  
  
**改为：**  
  
```python  
    # Decide dtype & attention implementation  
    if args.device == "mps":  
        load_dtype = torch.float16  # float16 比 float32 快且省内存，MPS 支持  
        attn_impl_primary = "sdpa"  # flash_attention_2 not supported on MPS  
```  
  
---  
  
找到 MPS 的 `from_pretrained` 调用（约第 205-213 行）：  
  
```python  
        if args.device == "mps":  
            model = VibeVoiceForConditionalGenerationInference.from_pretrained(  
                args.model_path,  
                torch_dtype=torch.float16,  
                attn_implementation=attn_impl_primary,  
                device_map=None,  # load then move  
                attn_implementation='sdpa'   # ← 重复了！  
            )  
            model.to("mps")  
```  
  
**改为：**  
  
```python  
        if args.device == "mps":  
            model = VibeVoiceForConditionalGenerationInference.from_pretrained(  
                args.model_path,  
                torch_dtype=load_dtype,       # 统一用上面定义的 float16  
                device_map=None,              # 先加载到 CPU，再 .to('mps')  
                attn_implementation='sdpa',  
            )  
            model.to("mps")  
            torch.mps.synchronize()  
            print("✅ Model moved to MPS (float16)")  
```  
  
---  
  
### 修改 2：`set_ddpm_inference_steps` 改成可配置  
  
找到这行（约第 248 行）：  
  
```python

```

```python  
    model.set_ddpm_inference_steps(num_steps=args.num_steps)

```
