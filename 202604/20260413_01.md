## 安装 Hermes Agent  
  
参考官网文档进行配置, 非常简单.  
  
https://hermes-agent.nousresearch.com/docs/user-guide/docker  
  
1. 先拉取镜像  
  
```  
docker pull nousresearch/hermes-agent  
```  
  
如果你遇到无法拉取的问题, 可用配置 docker desktop engine 设置使用 mirrors .  
```  
{  
  "builder": {  
    "gc": {  
      "defaultKeepStorage": "20GB",  
      "enabled": true  
    }  
  },  
  "experimental": false,  
  "registry-mirrors": [  
    "https://docker.1ms.run",  
    "https://docker.1panel.live",  
    "https://docker.xuanyuan.me",  
    "https://docker.m.daocloud.io"  
  ]  
}  
```  
  
或者使用proxy, 参考:  
- [《squid proxy》](../202407/20240704_01.md)  
  
  
2. 配置容器, 指定模型和key  
  
容器是无状态的, 所有产生的数据、配置文件等动态信息, 都将保留在容器内的 `/opt/data` 目录中, 所以只需要把这个目录映射到宿主机某个目录即可.  
  
例如 `~/.hermes` , 以后如果更换电脑, 只需要把这个目录拷贝过去即可.  
  
```  
mkdir -p ~/.hermes  
docker run -it --rm \
  -v ~/.hermes:/opt/data \
  nousresearch/hermes-agent setup  
```  
  
  
从 provider 列表中选择 **"MiniMax China (mainland China endpoint)"**。  
  
  
输入从 [MiniMax Token Plan 页面](https://platform.minimaxi.com/user-center/payment/token-plan)获取的 **Token Plan API Key**。  
  
  
选择 **MiniMax-M2.7** 作为模型。  
  
  
3. 配置微信  
  
启动 cli 模式, 直接在对话中配置微信  
```  
docker run -it --rm \
  -v ~/.hermes:/opt/data \
  nousresearch/hermes-agent  
```  
  
输入配置微信, 根据提示, 即可完成微信私人助理配置. 可能会要求微信客户端是最新版本, 其间可能需要你扫码授权等操作.  
  
设置完成后会有一些配置写入 `~/.env` 例如:  
```
WEIXIN_ACCOUNT_ID=xxxx@im.bot
WEIXIN_TOKEN=xxxx@im.bot:xxxx
WEIXIN_BASE_URL=https://ilinkai.weixin.qq.com
WEIXIN_CDN_BASE_URL=https://novac2c.cdn.weixin.qq.com/c2c
WEIXIN_DM_POLICY=pairing

WEIXIN_GROUP_POLICY=allowall
```
  
这个私人助理是你和 Hermes 沟通的私人消息通道, 它不能加入到任何微信群里.  
  
使用 Hermes gateway 模式时, 你可以通过这个私人助理给 Hermes 派活.  
  
4. 启用 Hermes gateway 模式.  
  
```  
docker run -d \
  --name hermes \
  --restart unless-stopped \
  -v ~/.hermes:/opt/data \
  nousresearch/hermes-agent gateway run  
```  
  
关闭  
  
```  
docker stop hermes  
```  
  
启动/重新启动  
  
```  
docker start hermes  
```  
  
  
5. 科学上网问题  
  
[《socks 代理》](../202310/20231029_01.md)  
  
[《http 代理》](../202407/20240704_01.md)  
  
如果你的 hermes 遇到了墙的问题, 可用在对话中告诉他你有代理, 先正常访问, 如果访问不通, 再启用代理:  
```  
all_proxy=socks5h://host.docker.internal:11112  
http_proxy=http://host.docker.internal:22222  
https_proxy=http://host.docker.internal:22222  
```  
  
而无需将它设置为 docker 容器的环境变量, 因为设置为环境变量就全部都走代理了.  
  
如果你发现有时 AI 挺傻的, 总是无法正确使用代理, 再考虑配置全局 docker 容器的环境变量, 方法:  
  
```
vi ~/.hermes/.env
  
HTTP_PROXY=http://host.docker.internal:22222
HTTPS_PROXY=http://host.docker.internal:22222
ALL_PROXY=http://host.docker.internal:22222
all_proxy=http://host.docker.internal:22222
http_proxy=http://host.docker.internal:22222  
https_proxy=http://host.docker.internal:22222  
```
  
或在 docker run 时设置环境变量
  
```
-e HTTP_PROXY=http://host.docker.internal:22222  \
-e HTTPS_PROXY=http://host.docker.internal:22222  \
-e ALL_PROXY=http://host.docker.internal:22222  \
-e all_proxy=http://host.docker.internal:22222  \
-e http_proxy=http://host.docker.internal:22222  \
-e https_proxy=http://host.docker.internal:22222  \
```

  
6. 如何确认   
  
使用微信安排任务时, 有时 Hermes 需要你授权, 你按需回复即可, 例如回复 `/approve`   
```
Reply /approve to execute, /approve session to approve this pattern for the session, /approve always to approve permanently, or /deny to cancel.
```

8. 升级镜像  
  
```
docker pull nousresearch/hermes-agent
```
  
删除使用老镜像启动的容器  
  
```
docker stop hermes
docker rm hermes
```
  
删除老镜像  
```
$ docker images
REPOSITORY                                                     TAG             IMAGE ID       CREATED          SIZE
nousresearch/hermes-agent                                      latest          5dd78ab86b23   30 minutes ago   8.1GB
<none>                                                         <none>          78ab91f60f08   2 days ago       7.17GB


docker rmi 78ab91f60f08
```
  
  
使用新镜像, 状态文件使用老的即可, 不需要额外的配置.  
```
docker run -d \
  --name hermes \
  --restart unless-stopped \
  -v ~/.hermes:/opt/data \
  nousresearch/hermes-agent gateway run  
```
  
如果遇到问题需要配置, 也可以  
```
docker run -it --rm \
  -v ~/.hermes:/opt/data \
  nousresearch/hermes-agent setup  
```
  
9. 你也可以直接进入 gateway 的容器执行一些操作.  例如  
```
docker exec -ti hermes bash

# 查看 gateway 是如何启动的, 照它的命令使用 hermes 即可.
top -c
/opt/hermes/.venv/bin/python3 /opt/hermes/.venv/bin/hermes gateway run

/opt/hermes/.venv/bin/python3 /opt/hermes/.venv/bin/hermes doctor --fix
```
  
  
10. 其他  
  
10\.1、如果你查看 gateway 容器发现这样的告警  
```  
docker logs hermes  
  
WARNING gateway.run: No user allowlists configured.  
All unauthorized users will be denied.  
Set GATEWAY_ALLOW_ALL_USERS=true in ~/.hermes/.env to allow open access,  
or configure platform allowlists (e.g., TELEGRAM_ALLOWED_USERS=your_id).  
```  
  
可修改 `~/.hermes/.env`  
  
```  
GATEWAY_ALLOW_ALL_USERS=true  
```  
  
重启 Hermes Gateway 容器生效.  
  
⚠️ 注意：这会开放访问，任何人都能连接到你这个 Hermes 实例。如果只是自己用没问题；如果暴露在公网，建议配置具体的用户白名单。  
  
**强烈建议你不要把容器的端口暴露给公网.**  
  
10\.2、hermes 内部除了使用 `/opt/data`, 它自己的`HOME`目录可能是`/opt/hermes`, 例如有些文件可能会放在这个目录, 怎么搞出来? `docker cp`, 或进入容器后拷贝到 `/opt/data` 目录, 就可以在宿主机 `~/.hermes` 看到了.  
  
不管是临时 `chat` 容器还是 `gateway` 容器, 都可在宿主机上使用 `docker exec -ti 容器名 bash` 进入.
