## 原理详解

### 1. UPDATE/DELETE FOR PORTION OF

#### 1.1 语法

```sql
-- UPDATE FOR PORTION OF
UPDATE t FOR PORTION OF valid_at
    FROM '2001-01-01' TO '2002-01-01'
SET col = value;

-- DELETE FOR PORTION OF
DELETE FROM t FOR PORTION OF valid_at
    FROM '2001-01-01' TO '2002-01-01';
```

#### 1.2 适用条件

`valid_at` 列必须是：

- `DATERANGE` 或 `TSTZRANGE` 类型
- 或 `Datemultirange` / `Tstzmultirange` 类型

#### 1.3 实现原理（概述）

**解析层**：在 `src/backend/parser/parser_gram.y` 中添加 `FOR PORTION OF` 语法支持

**执行层**：核心处理逻辑在 `src/backend/executor/` 中的相关执行函数处理时间范围的重叠计算和行切分

**语义**：
1. 找出所有与目标时间段重叠的行
2. 对于部分重叠的行，切分历史记录
3. 更新/删除目标部分
4. 插入剩余部分（如有）

### 2. 图模式匹配 (Graph Pattern Matching)

#### 2.1 语法 (Cypher 风格)

```sql
-- 节点模式
SELECT *
FROM MATCH (a:Person)-[:KNOWS]->(b:Person)
WHERE a.name = 'Alice';

-- 属性引用
SELECT *
FROM MATCH (a {name: 'Alice'})-[:KNOWS]->(b)
WHERE a.age > 30;
```

#### 2.2 解析器支持

**解析器**：`src/backend/parser/parser_gram.y`

在 `CREATE TABLE` 等语句中添加 `GRAPH` 关键字支持，在 `SELECT` 语句中添加 `MATCH` 关键字支持。

### 3. 属性引用优先

#### 3.1 原理 (`c5b3253b8abd`)

在图模式中，`{name: 'Alice'}` 优先解析为**图的属性**，而不是**列引用**：

```sql
-- 之前：可能误解为查找名为 'name' 的列
(a {name: 'Alice'})

-- 现在：明确为图属性引用
```

### 4. 变量只添加一次

#### 4.1 原理 (`288ae968726e`)

同一个变量在模式中多次出现应该指向同一个节点：

```sql
-- a 应该指向同一个节点
MATCH (a)-[:KNOWS]->(b)-[:KNOWS]->(a)
-- 这表示 a 认识 b，b 认识 a，形成闭环
```

### 5. 拒绝连续同类型模式

#### 5.1 原理 (`720f0f89d696`)

连续的两个关系模式应该被拒绝：

```sql
-- 非法：两个连续的 -> 关系
(a)-[:KNOWS]->(b)->[:LIKES]->(c)

-- 应该：
(a)-[:KNOWS]->(b)-[:LIKES]->(c)
```

## 使用实践

### 1. 时态表基础

#### 1.1 创建时态表

```sql
-- 使用 daterange 列
CREATE TABLE employee (
    id SERIAL PRIMARY KEY,
    name TEXT,
    valid_at DATERANGE,
    position TEXT
);

-- 使用排除约束确保不重叠
ALTER TABLE employee
    ADD CONSTRAINT no_overlap
    EXCLUDE USING GIST (valid_at WITH &&);
```

#### 1.2 插入时态数据

```sql
INSERT INTO employee (name, valid_at, position) VALUES
    ('Alice', '[2020-01-01, 2022-01-01)', 'Engineer'),
    ('Alice', '[2022-01-01, )', 'Senior Engineer');
```

#### 1.3 时态查询

```sql
-- 查询特定时间点的数据
SELECT * FROM employee
WHERE valid_at @> '2021-06-15';

-- 查询历史状态
SELECT * FROM employee
WHERE valid_at @> '2021-06-15' AND valid_at <@ '[2020-01-01, 2022-01-01)';
```

### 2. FOR PORTION OF 更新

#### 2.1 基本用法

```sql
-- 更新 Alice 在 2021 年的职位
UPDATE employee FOR PORTION OF valid_at
    FROM '2021-01-01' TO '2022-01-01'
SET position = 'Lead Engineer'
WHERE name = 'Alice';
```

#### 2.2 结果

```
之前：
| name  | valid_at              | position       |
|-------|-----------------------|----------------|
| Alice | [2020-01-01, 2022-01-01) | Engineer     |

UPDATE FOR PORTION OF 后：
| name  | valid_at              | position       |
|-------|-----------------------|----------------|
| Alice | [2020-01-01, 2021-01-01) | Engineer     |
| Alice | [2021-01-01, 2022-01-01) | Lead Engineer |
```

### 3. 图模式匹配

#### 3.1 基本图查询

```sql
-- 查找直接朋友
SELECT a.name, b.name
FROM MATCH (a:Person)-[:KNOWS]->(b:Person)
WHERE a.name = 'Alice';

-- 查找朋友的朋友（两跳）
SELECT a.name, c.name
FROM MATCH (a:Person)-[:KNOWS]->(b:Person)-[:KNOWS]->(c:Person)
WHERE a.name = 'Alice' AND c.name != 'Alice';
```

#### 3.2 可变长度路径

```sql
-- 查找任意长度的 KNOWS 路径
SELECT *
FROM MATCH (a:Person)-[:KNOWS*1..3]->(b:Person)
WHERE a.name = 'Alice';
```

#### 3.3 属性过滤

```sql
-- 查找特定属性的节点
SELECT *
FROM MATCH (p:Person {name: 'Alice', age: > 30})-[:KNOWS]->(friend)
```

### 4. 性能考虑

#### 4.1 时态表索引

```sql
-- 为时间范围列创建 GiST 索引
CREATE INDEX idx_employee_valid_at ON employee USING GIST (valid_at);

-- 复合索引
CREATE INDEX idx_employee_name_valid ON employee (name, valid_at);
```

#### 4.2 图查询索引

```sql
-- 为标签创建索引
CREATE INDEX idx_person_name ON Person (name);
CREATE INDEX idx_person_age ON Person (age);

-- 为关系类型创建索引
CREATE INDEX idx_knows_from ON KNOWS (from_node);
CREATE INDEX idx_knows_to ON KNOWS (to_node);
```

### 5. 限制与注意事项

#### 5.1 FOR PORTION OF 限制

```sql
-- 只支持 DATERANGE/TSTZRANGE 列
-- 必须是不重叠的表

-- 以下情况不支持：
-- - 分区表
-- - 有多个时间范围列
-- - 有时态约束的表
```

#### 5.2 图模式限制

```sql
-- 目前只支持简单的路径模式
-- 不支持递归 CTEs 的图查询
-- 不支持属性图的全部特性
```

## 总结

时态表和图模式匹配是 PostgreSQL 对 SQL:2023 标准的重要支持。

**关键要点**：

1. **FOR PORTION OF**：基于时间范围的 UPDATE/DELETE
2. **自动切分**：部分重叠时自动创建历史记录
3. **图模式匹配**：Cypher 风格的图查询
4. **属性优先**：图属性引用优先于列引用

**适用场景**：

1. **时态数据**：历史追踪、审计日志
2. **图数据**：社交网络、推荐系统
3. **业务时间**：保险、医疗、HR 系统
