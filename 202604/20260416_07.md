## 背景  
  
这周发生的事情，可能会让很多人重新想想：私人AI助理，到底该不该自己折腾？  
  
---  
  
**先说说这两个最近有多火**  
  
如果你在开发者社区混，最近一定见过两个名字刷屏 —— OpenClaw 和 Hermes。  
  
OpenClaw 是奥地利开发者 Peter Steinberger 在2025年11月发布的开源个人AI助理框架，成了今年增长最快的开源项目之一。它的核心卖点是数据主权：你的文件和对话都留在本地机器上，不会跑到别人的服务器去。  
  
你可以在 WhatsApp 上发条消息，它就帮你查邮件、改代码、开PR——就像雇了个人坐在电脑旁边，只不过这个人永远不需要喝咖啡。  
  
Hermes 是后来者，由 Nous Research 在2026年2月发布，但势头丝毫不弱。它的标语是"与你一同成长的助理"，是目前唯一一个具备闭环自我学习能力的开源框架——它解决完任务之后，会自己写一份可复用的技能文档，记录经验，下次遇到类似问题时更快更准。  
  
两个项目都在社区里引发了巨大反响，用户们的使用案例越来越疯狂——自动处理邮件、监控服务器、管理客户跟进、甚至帮着做税务申报。  
  
看起来一片欣欣向荣。  
  
直到今天。  
  
---  
  
**原厂出手了** 其实不仅仅是 Claude, 所有顶级大模型厂商都可以干这件事. 其实我之前就预言过 [《德说-第462期, 别着急学 OpenClaw，它即将被取代》](../202603/20260324_02.md)  
  
今天，Anthropic 同时做了三件事：  
  
第一，Claude Code 桌面端彻底重构。同一窗口内并行跑多个 Claude、内置终端、原生文件编辑、SSH远程连接 —— 以前需要在五六个工具之间反复横跳的事，现在一个窗口全搞定。  
  
第二，Routines 功能正式上线。定时触发、API 触发、GitHub 事件触发，三路同时开放。配好一次，Claude 在云端自动帮你干活，不依赖你的电脑开着。  
  
第三，也是最重磅的 —— Claude Opus 4.7 今天正式发布了。  
  
这不是小更新。与此同时，Anthropic 还推出了一款全新的 AI 设计工具，可以直接生成网站、演示文稿和产品视觉原型，消息一出，Figma、Adobe、Wix 股价当天跌幅均超过2%。  
  
但对于 OpenClaw 和 Hermes 的用户来说，真正值得停下来想一想的，是 Routines 这个东西。  
  
---  
  
**Routines 到底动了谁的蛋糕**  
  
OpenClaw 和 Hermes 解决的核心问题，说穿了就一个：让 AI 替你持续干活，不用你一直盯着。  
  
OpenClaw 把 AI 助理接进 WhatsApp 和 Telegram，你发条消息就能触发任务。Hermes 更激进，任务完成之后还会自我学习，越用越聪明。  
  
这两套方案的共同代价是：它们的配置相当复杂，涉及命令行操作和大量技术细节；如果你不懂怎么跑命令，这类项目对你来说就太危险了。  
  
另外还有一个始终绕不开的问题——安全。思科的 AI 安全研究团队测试了一个第三方 OpenClaw 技能包，发现它在用户不知情的情况下进行了数据泄露和提示注入，该技能仓库缺乏足够的审查机制来防范恶意提交。  
  
而 Routines 呢？  
  
你在 Anthropic 的界面里点几下，设好提示词、触发器和云端环境，Claude 就在 Anthropic 的基础设施上跑。不需要自己搭 Docker 镜像，不需要懂命令行，不需要担心进程挂掉、电脑睡眠、网络断线。  
  
从功能上说，Routines 几乎覆盖了 OpenClaw 和 Hermes 80% 的核心使用场景——定时任务、API 触发、GitHub 事件响应——而且开箱即用。  
  
如果你是普通用户，这个对比已经很残忍了。  
  
---  
  
**那私人助理们还有没有活路**  
  
有。但只剩一条路了。  
  
OpenClaw 的 GitHub 页面上有一句话，被很多早期用户反复引用：OpenClaw 最大的差异化在于数据所有权——它运行在你自己的机器上，你的基础设施，你的密钥，你的数据。  
  
这句话在今天依然成立。  
  
Routines 再好用，你的数据是在 Anthropic 的云端跑的。对于有数据合规要求的企业、对隐私极度敏感的个人用户，或者需要接入私有内网系统的场景，私有部署依然不可替代。  
  
OpenClaw 和 Hermes Agent 代表了2026年个人AI助理的两种主流路径：OpenClaw 专注于集成广度和手动控制，而 Hermes 专注于记忆深度与自我进化。这种深度定制和本地化能力，是云端产品短期内给不了的。  
  
但问题是，选择私有部署的门槛实实在在地摆在那里。你需要懂技术，需要自己维护，需要承担安全风险。  
  
原厂下场之后，留给这类工具的空间，就是那些愿意为"数据不出门"付出额外代价的用户。  
  
这个群体真实存在，但比之前想象的要小得多。  
  
---  
  
**最后说一句扎心的**  
  
过去几个月，技术社区流行一种浪漫叙事：用开源工具，自己搭私人助理，数字主权在我不在云。  
  
这个叙事没有错。但它成立的前提，是原厂没有认真做同类产品。  
  
现在原厂认真了。  
  
Opus 4.7 上线，Routines 上线，设计工具上线，Claude Code 重构——Anthropic 正在把"AI替你干活"这件事，变成一个普通人也能用的产品，而不是极客的专属玩具。  
  
OpenClaw 和 Hermes 还会存在，也还会有人用。  
  
但它们的天花板，今天已经画出来了：私有部署环境。  
  
走出这个范围，它们面对的，是一个有无限算力、有原生模型、有完整工程团队支撑的对手。  
  
大模型原厂下场，从来不温柔。
