## 一句话理解：lark-cli 是什么？  
  
**lark-cli** 是飞书官方出品的命令行工具，让你（或者 AI）可以像发指令一样操作飞书——发消息、查日历、写文档、搜邮件……一行命令搞定，不需要打开 App。它同时为 AI Agent 量身设计，是目前最完整的飞书自动化开源方案之一（GitHub 900+ Star，MIT 协议）。  
  
[lark-cli 项目地址](https://github.com/larksuite/cli)  
  
## lark-cli 解决了什么问题？  
  
飞书有海量 API，但直接调用需要处理鉴权、分页、参数拼装等大量细节。lark-cli 把这些都包装好了，提供三层抽象：  
  
| 层级 | 面向谁 | 举例 |  
|------|--------|------|  
| **快捷命令**（`+`前缀） | 人 & AI，开箱即用 | `lark-cli calendar +agenda` |  
| **API 命令** | 精确控制，100+ 接口 | `lark-cli calendar events instance_view --params '...'` |  
| **原始 API** | 开发者，覆盖 2500+ 接口 | `lark-cli api GET /open-apis/...` |  
  
  
  
## 实践：3 分钟上手  
  
**第一步：安装**  
```bash  
npm install -g @larksuite/cli  
npx skills add larksuite/cli -y -g   # 安装 AI Agent 技能包  
```  
  
**第二步：配置 & 登录**  
```bash  
lark-cli config init     # 引导创建飞书应用（浏览器完成）  
lark-cli auth login --recommend   # 一键授权常用权限  
```  
  
**第三步：用起来！**  
```bash  
# 看今天的日程  
lark-cli calendar +agenda  
  
# 给某个群发消息  
lark-cli im +messages-send --chat-id "oc_xxx" --text "晚上开会提醒"  
  
# 创建一篇文档  
lark-cli docs +create --title "周报" --markdown "# 本周进展\n- 完成了功能X"  
  
# 预览（不真正执行）  
lark-cli im +messages-send --chat-id oc_xxx --text "hello" --dry-run  
```  
  
  
  
## 它特别适合哪些场景？  
  
**场景一：AI 助手自动化**  
接入 Claude / GPT 等 AI 工具后，AI 可以直接帮你"操作"飞书——比如自动整理会议纪要发到群里、每天定时发日报、从日历中提取待办事项。19 个现成的 Agent Skills 无需额外开发，直接拿来用。  
  
**场景二：开发者脚本**  
配合 Shell 脚本或 CI/CD，实现自动化通知、表格数据写入、文件上传等流程。  
  
**场景三：快速查询**  
不想打开 App 翻日历？一行命令查今天的会议，结果可以输出为 JSON、表格、CSV，方便二次处理。  
  
  
  
## 安全性值得特别关注  
  
由于 AI 会以**你的用户身份**操作飞书，lark-cli 内置了多层防护：输入注入拦截、终端输出脱敏、系统级密钥存储（不明文保存）。官方建议：**不要把授权后的 Bot 加到群聊里**，避免他人滥用权限。  
  
  
  
总体来说，lark-cli 是一个"一次配置、终身受益"的工具——对普通用户是效率利器，对 AI Agent 是现成的飞书操作能力，对开发者是完整的自动化脚手架。
