## 背景    
  
Mac M2 能跑 windows x86_64 虚拟机吗?  
  
架构不同：Mac M2芯片基于ARM架构，而Windows 7是为x86架构设计的。这就像试图把汽油车的发动机装到电动车上，虽然都是车，但核心原理不通用.   
  
如果核心需求就是运行x86 Windows 7，UTM是目前最值得尝试的免费工具，它对x86系统的模拟支持更好. UTM 基于QEMU的强大模拟器，可以在ARM Mac上模拟x86架构。免费且开源；专门为Apple Silicon优化；对x86系统的模拟能力比VirtualBox更成熟；你甚至可以用它来模拟更老的系统，如Windows XP. 模拟的性能损失依然存在，肯定不如原生虚拟化流畅，但对于运行一些老的、对性能要求不高的软件来说，通常够用。

附smb配置:  
  
1.  **Mac 端设置（开启文件共享）** ：  
    *   打开 Mac 的 **"系统设置" (System Settings)** ，点击 **"通用" (General)** -> **"共享" (Sharing)** 。  
    *   打开 **"文件共享" (File Sharing)** 开关 。  
    *   你可以通过下方的 **"选项..." (Options...)** 按钮，确保勾选了 **"使用 SMB 来共享文件和文件夹" (Share files and folders using SMB)** 。  
    *   点击信息图标 (i) 或记下屏幕上方显示的 Mac 局域网地址，例如 `smb://192.168.1.100`。你需要的是 `192.168.1.100` 这部分 IP 地址 。  
  
2.  **Windows 7 端设置（访问共享）** ：  
    *   启动 Windows 7 虚拟机。  
    *   确保虚拟机的网络模式设置正确。在 UTM 虚拟机设置里，网络设备通常选 **"桥接 (Bridge)"** 或 **"共享网络 (Shared Network)"** 都可以 。桥接模式下，Win7 和 Mac 就在同一个局域网段，更容易互相发现。  
    *   在 Win7 中，打开 **"计算机" (Computer)** ，点击顶部工具栏的 **"映射网络驱动器" (Map network drive)** 。  
    *   **"驱动器" (Drive)** 任选一个盘符。在 **"文件夹" (Folder)** 框中，输入 `\\你刚才记下的Mac IP地址\你的Mac用户名`。例如，如果你的 Mac IP 是 `192.168.1.100`，用户名叫 `apple`，就输入 `\\192.168.1.100\apple` 。  
    *   点击 **"完成" (Finish)** 。这时可能会弹出登录窗口，输入你的 **Mac 的系统登录密码** 即可。  
    *   之后，你的 Mac 个人文件夹就会作为一个网络驱动器出现在 Win7 的“计算机”里了。你可以在里面自由读写文件 。
