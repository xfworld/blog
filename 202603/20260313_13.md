## 背景          
  
### 11.4.1 向量存储基础  
  
在现代数据库中，向量存储（Vector Storage）成为支撑 AI、推荐系统、图像检索和自然语言处理等应用的关键技术。PostgreSQL 的扩展插件 pgvector 提供了在关系型数据库中直接管理向量的能力，实现向量与传统结构化数据的无缝融合。  
  
#### 1. 向量数据及其价值  
  
向量是由连续数值组成的高维数据表示（embedding），可用于表示文本、图像、音频等对象的特征。通过向量化表示，可以将复杂对象的语义信息压缩为定长的浮点数组，从而在数据库中进行快速相似度搜索。  
  
应用案例：  
  
* 自然语言处理：将文章或句子向量化，实现语义搜索。  
* 图像检索：用 CNN 提取图像特征向量，进行内容相似度查询。  
* 推荐系统：将用户和商品映射到向量空间，实现近邻匹配。  
  
#### 2. pgvector 插件概览  
  
pgvector 是 PostgreSQL 的开源扩展，提供向量列类型、距离计算函数以及 ANN（Approximate Nearest Neighbor）索引支持。它核心特性包括：  
  
* 向量类型：`vector`（float32）、`halfvec`（float16）、`sparsevec`（稀疏向量）、`bit`（二进制量化）。  
* 距离函数：支持 L2、内积（inner product）、余弦相似度（cosine）、L1、Hamming、Jaccard 等。  
* 索引方法：HNSW（Hierarchical Navigable Small World）、IVFFlat（Inverted File with Flat quantization）。  
* 完全集成 PostgreSQL：ACID、WAL 复制、支持任意客户端语言。  
  
> 案例数据：某 AI 文本搜索平台通过 pgvector 将文本向量存储在 PostgreSQL 中，并利用 HNSW 索引实现每秒 10,000+ 查询吞吐量，同时维持数据一致性和备份能力。  
  
#### 3. 向量类型与存储  
  
pgvector 提供四种向量类型，各有特性：  
  
| 类型          | 数据格式                | 优势               | 典型应用       |  
| ----------- | ------------------- | ---------------- | ---------- |  
| `vector`    | float32             | 精度高，兼容性好         | NLP、图像特征   |  
| `halfvec`   | float16             | 内存占用低，支持 F16C 加速 | 大规模向量库     |  
| `sparsevec` | `{index:value,...}` | 稀疏数据存储节省空间       | 文本特征向量     |  
| `bit`       | 二进制量化               | 高速近似搜索           | 大规模 ANN 查询 |  
  
存储示例：  
  
```sql  
CREATE TABLE items (  
    id serial PRIMARY KEY,  
    embedding vector(1536)  
);  
INSERT INTO items (embedding) VALUES ('[0.12, 0.34, ..., 0.56]');  
```  
  
#### 4. 距离计算与相似度搜索  
  
pgvector 内置距离操作符，使 SQL 查询可以直接完成向量相似度计算：  
  
* `<->`：L2 距离  
* `<#>`：内积  
* `<=>`：余弦相似度  
  
查询示例：  
  
```sql  
SELECT id, embedding <-> '[0.1,0.2,0.3]' AS distance  
FROM items  
ORDER BY distance  
LIMIT 5;  
```  
  
#### 5. 索引与性能优化  
  
为了处理大规模向量，pgvector 提供两种 ANN 索引：  
  
* HNSW：适合高维、在线增量更新，搜索速度快。  
* IVFFlat：适合批量构建、离线索引，内存占用可控。  
  
索引示例：  
  
```sql  
CREATE INDEX ON items USING hnsw (embedding vector_l2_ops);  
```  
  
#### 小结  
  
向量存储基础不仅是 AI 数据处理的核心，也是多模数据库的关键能力。通过 pgvector，PostgreSQL 可以：  
  
1. 将向量直接融入关系型数据。  
2. 提供精确与近似相似度搜索。  
3. 支持多种索引与距离计算，兼顾性能与存储效率。
