## 背景                    
盆友说可以搞个企业邮箱, 企业试用 chatgpt business 版一个月.   
  
也不知道真假, 反正把我的 chatgpt 账号拉到 team 了, 现在可以不用 api key 直接使用 chatgpt 账号来使用 codex , 包括网页版本的 chatgpt 也升级到商业版了.   
  
不过加入过程要全程使用代理, chrome 的纯 proxy 模式都不行, 最后调整了 macOS 的网络配置, 把 socks/http/https 代理全设置一遍, 可以了.

codex 客户端使用参考:   
  
https://github.com/openai/codex  
  
似乎在安装了 macOS 版 codex APP 后, 如果配置好 codex APP 的认证, 再安装 codex cli 的话, 会自动配置 `~/.codex/auth.json`  
```  
cat auth.json   
{  
  "auth_mode": "chatgpt",  
  "OPENAI_API_KEY": null,  
  "tokens": {  
    "id_token": "  
    ......  
```  
  
如果你的 codex cli 无法连接模型(例如prompt后, 一直处于时间流逝状态), 可能要在启动时加入proxy配置.    
```  
mkdir ~/workspace_codex  
cd ~/workspace_codex  
all_proxy=socks5h://127.0.0.1:11112 http_proxy=http://127.0.0.1:22222 https_proxy=http://127.0.0.1:22222 codex  
  
# 假设你已经配置了如上代理  
```  
  
**同理, 使用这个方式启动 codex APP:**   
```  
mkdir ~/workspace_codex  
cd ~/workspace_codex  
all_proxy=socks5h://127.0.0.1:11112 http_proxy=http://127.0.0.1:22222 https_proxy=http://127.0.0.1:22222 nohup /Applications/Codex.app/Contents/MacOS/Codex >/dev/null 2>&1 &  
  
# 假设你已经配置了如上代理  
```
