## 背景               
### 11.2.3 图数据库插件的安装和配置  
  
在 PostgreSQL 生态中，实现图数据库能力最成熟的扩展之一是 Apache AGE。  
Apache AGE（A Graph Extension）是一个开源 PostgreSQL 扩展，它为数据库提供属性图模型（Property Graph Model）和 openCypher 查询语言支持，使 PostgreSQL 能够同时支持关系数据和图数据。  
  
Apache AGE 由韩国数据库公司 Bitnine 发起，并在 2022 年进入 Apache Software Foundation 生态，成为 Apache 开源项目之一。该扩展通过 PostgreSQL 的扩展框架实现，与 PostgreSQL 的事务系统、存储引擎和 SQL 查询引擎深度集成。   
  
本节将介绍 Apache AGE 的安装流程、配置方法以及基础验证步骤。  
  
   
  
#### 一、安装环境要求  
  
在安装 Apache AGE 之前，需要满足 PostgreSQL 运行环境和依赖库要求。  
  
##### 1 PostgreSQL版本要求  
  
Apache AGE 与 PostgreSQL 内核版本存在兼容关系。  
当前版本支持的 PostgreSQL 版本包括：  
  
* PostgreSQL 11  
* PostgreSQL 12  
* PostgreSQL 13  
* PostgreSQL 14  
* PostgreSQL 15  
* PostgreSQL 16  
* PostgreSQL 17    
* PostgreSQL 18    
  
因此，在安装 AGE 扩展前，应确认 PostgreSQL 版本：  
  
```bash  
pg_config --version  
```  
  
如果版本不兼容，需要改成兼容的 PostgreSQL 版本。  
  
   
  
##### 2 系统依赖库  
  
编译 Apache AGE 需要一些基础开发工具和系统库。  
在 Linux 系统中常见依赖包括：  
  
| 依赖库      | 作用      |  
| -------- | ------- |  
| gcc      | C语言编译器  |  
| readline | 交互式命令支持 |  
| zlib     | 压缩库     |  
| flex     | 词法分析器   |  
| bison    | 语法分析器   |  
  
例如在 Ubuntu 系统中：  
  
```bash  
sudo apt-get install build-essential \
libreadline-dev zlib1g-dev flex bison  
```  
  
这些工具用于构建 PostgreSQL 扩展模块。    
  
   
  
#### 二、从源码安装 Apache AGE  
  
目前 Apache AGE 最常见的安装方式是从源码编译安装。  
  
##### 1 获取源码  
  
首先从 GitHub 下载源码：  
  
```bash  
git clone https://github.com/apache/age.git  
cd age  
```  
  
源码仓库包含：  
  
| 文件          | 作用     |  
| ----------- | ------ |  
| Makefile    | 构建脚本   |  
| age.control | 扩展定义文件 |  
| SQL脚本       | 创建扩展对象 |  
| C源码         | 图数据库实现 |  
  
   
  
##### 2 编译并安装扩展  
  
在源码目录中执行：  
  
```bash  
make install  
```  
  
该命令会调用 PostgreSQL 扩展构建系统 PGXS（PostgreSQL Extension Build System），并完成以下操作：  
  
1. 编译 AGE C 代码  
2. 生成动态库 `age.so`  
3. 安装 SQL 扩展脚本  
4. 注册扩展控制文件  
  
如果 PostgreSQL 安装路径未在 PATH 中，可以指定 `pg_config`：  
  
```bash  
make PG_CONFIG=/usr/pgsql-18/bin/pg_config install  
```  
  
`pg_config` 用于提供 PostgreSQL 的安装路径信息。   
  
安装完成后，扩展文件通常位于：  
  
```text  
$PGHOME/lib/age.so  
$PGHOME/share/extension/age.control  
```  
  
   
  
#### 三、数据库级启用扩展  
  
扩展安装完成后，需要在数据库中启用。  
  
连接 PostgreSQL：  
  
```bash  
psql -U postgres  
```  
  
执行：  
  
```sql  
CREATE EXTENSION age;  
```  
  
该命令会创建 AGE 的系统对象，包括：  
  
* 图数据类型 `agtype`  
* Cypher 查询函数  
* 图管理函数  
* 系统 schema `ag_catalog`  
  
AGE 的系统对象主要存放在 `ag_catalog` schema 中。   
  
   
  
#### 四、会话级加载扩展  
  
与部分 PostgreSQL 扩展不同，Apache AGE 需要在每个会话中加载模块。  
  
执行：  
  
```sql  
LOAD 'age';  
```  
  
随后建议修改 `search_path`：  
  
```sql  
SET search_path = ag_catalog, "$user", public;  
```  
  
这样可以简化 Cypher 查询语法。  
  
例如：  
  
```sql  
SELECT *   
FROM cypher('graph', $$ MATCH (n) RETURN n $$)  
AS (v agtype);  
```  
  
如果没有设置 `search_path`，则需要显式写出 `ag_catalog.cypher`。  
  
   
  
#### 五、Docker方式安装（推荐测试环境）  
  
为了降低部署复杂度，Apache AGE 还提供了官方 Docker 镜像。  
  
首先拉取镜像：  
  
```bash  
docker pull apache/age  
```  
  
然后启动容器：  
  
```bash  
docker run \
--name age \
-p 5455:5432 \
-e POSTGRES_USER=postgresUser \
-e POSTGRES_PASSWORD=postgresPW \
-e POSTGRES_DB=postgresDB \
-d apache/age 
```  
  
进入数据库：  
  
```bash  
docker exec -it age psql -U postgresUser -d postgresDB  
```  
  
该镜像已经预安装：  
  
* PostgreSQL  
* Apache AGE  
* 扩展初始化脚本  
  
因此非常适合用于：  
  
* 图数据库学习  
* 测试环境  
* 教学实验环境。   
  
   
  
#### 六、验证安装是否成功  
  
完成安装后，可以创建一个简单图进行测试。  
  
##### 1 创建图  
  
```sql  
SELECT create_graph('social_graph');  
```  
  
该函数会创建一个新的图数据库空间。  
  
   
  
##### 2 创建节点  
  
```sql  
SELECT *  
FROM cypher('social_graph', $$  
    CREATE (n:User {name:'Alice'})  
$$) AS (v agtype);  
```  
  
   
  
##### 3 查询节点  
  
```sql  
SELECT *  
FROM cypher('social_graph', $$  
    MATCH (n:User)  
    RETURN n  
$$) AS (v agtype);  
```  
  
如果能够返回节点数据，则说明 Apache AGE 已成功安装。  
  
   
  
#### 七、生产环境部署建议  
  
在生产环境中部署 Apache AGE 时，通常需要注意以下事项：  
  
##### 1 PostgreSQL版本匹配  
  
由于 AGE 依赖 PostgreSQL 内核接口，不同版本之间可能存在 ABI 差异，因此需要确保：  
  
* PostgreSQL版本  
* AGE版本  
  
之间保持兼容。  
  
   
  
##### 2 扩展权限控制  
  
默认情况下，只有超级用户可以加载扩展。  
  
可以通过以下方式允许普通用户使用：  
  
```sql  
GRANT USAGE ON SCHEMA ag_catalog TO app_user;  
```  
  
同时可以通过符号链接允许非超级用户加载扩展库。   
  
   
  
##### 3 资源配置  
  
图查询通常具有以下特点：  
  
* 大量路径遍历  
* 复杂模式匹配  
* 内存使用较高  
  
因此建议适当调优 PostgreSQL 参数：  
  
| 参数                   | 建议       |  
| -------------------- | -------- |  
| work_mem             | 提高       |  
| shared_buffers       | 根据内存调整   |  
| effective_cache_size | 设置为内存70% |  
  
   
  
#### 小结  
  
Apache AGE 是 PostgreSQL 生态中最重要的图数据库扩展之一。  
通过该扩展，PostgreSQL 可以同时支持：  
  
* 关系数据模型  
* 图数据模型  
* SQL 查询语言  
* Cypher 图查询语言  
  
Apache AGE 的安装主要包括以下步骤：  
  
1. 准备 PostgreSQL 环境  
2. 安装系统依赖  
3. 编译并安装 AGE 扩展  
4. 在数据库中启用扩展  
5. 配置会话环境  
  
完成这些步骤后，PostgreSQL 即可具备完整的图数据库能力。
