## 背景                 
### 11.2.4 图数据库插件的使用实践  
  
在完成 Apache AGE 扩展的安装与配置之后，PostgreSQL 即可具备图数据库能力。本节通过一个完整的实践案例，介绍如何使用 Apache AGE 在 PostgreSQL 中完成图数据建模与查询，包括：  
  
1. 图创建  
2. 节点与边的创建  
3. 图模式查询  
4. 路径与关系分析  
5. SQL 与图查询混合使用  
  
Apache AGE 使用 openCypher 查询语言，并通过 `cypher()` 函数嵌入 PostgreSQL SQL 查询体系，使图查询能够与关系查询协同工作。   
  
   
  
#### 一、图数据模型实践案例  
  
为了说明图数据库的实际应用，本节构建一个简单的社交网络图模型。  
  
该模型包含：  
  
节点（Vertex）  
  
| 标签      | 含义 |  
| ------- | -- |  
| User    | 用户 |  
| Company | 公司 |  
  
边（Edge）  
  
| 类型       | 含义   |  
| -------- | ---- |  
| FRIEND   | 好友关系 |  
| WORKS_AT | 工作关系 |  
  
图结构示例：  
  
```  
Alice ---FRIEND---> Bob  
  |                  |  
WORKS_AT          WORKS_AT  
  |                  |  
CompanyA         CompanyB  
```  
  
这种关系网络是图数据库最典型的应用之一，例如社交网络和推荐系统。  
  
   
  
#### 二、创建图空间  
  
在 Apache AGE 中，所有图数据都存储在一个 Graph Namespace 中。  
  
创建图：  
  
```sql  
SELECT create_graph('social_graph');  
```  
  
该命令会自动创建一组内部表，用于存储：  
  
* 节点  
* 边  
* 属性数据  
  
这些表由 AGE 自动管理。   
  
   
  
#### 三、创建节点（Vertex）  
  
在图数据库中，节点表示实体，例如用户或公司。  
  
##### 1 创建用户节点  
  
```sql  
SELECT *  
FROM cypher('social_graph', $$  
CREATE (:User {name:'Alice', age:30}),  
       (:User {name:'Bob', age:28}),  
       (:User {name:'Charlie', age:35})  
$$) AS (v agtype);  
```  
  
节点包含：  
  
* 标签（User）  
* 属性（name、age）  
  
在 Apache AGE 中，所有返回结果都使用 `agtype` 数据类型，这是 AGE 定义的一种扩展类型，类似 JSONB 的超集。   
  
   
  
##### 2 创建公司节点  
  
```sql  
SELECT *  
FROM cypher('social_graph', $$  
CREATE (:Company {name:'TechCorp'}),  
       (:Company {name:'DataAI'})  
$$) AS (v agtype);  
```  
  
现在图中已经包含两种实体：  
  
```  
User  
Company  
```  
  
   
  
#### 四、创建关系（Edge）  
  
在图数据库中，边表示实体之间的关系。  
  
##### 创建好友关系  
  
```sql  
SELECT *  
FROM cypher('social_graph', $$  
MATCH (a:User {name:'Alice'}),  
      (b:User {name:'Bob'})  
CREATE (a)-[:FRIEND]->(b)  
RETURN a,b  
$$) AS (a agtype, b agtype);  
```  
  
该语句包含三个部分：  
  
| 语法     | 作用   |  
| ------ | ---- |  
| MATCH  | 查找节点 |  
| CREATE | 创建关系 |  
| RETURN | 返回结果 |  
  
   
  
##### 创建工作关系  
  
```sql  
SELECT *  
FROM cypher('social_graph', $$  
MATCH (u:User {name:'Alice'}),  
      (c:Company {name:'TechCorp'})  
CREATE (u)-[:WORKS_AT]->(c)  
RETURN u,c  
$$) AS (u agtype, c agtype);  
```  
  
图结构变为：  
  
```  
Alice ----FRIEND----> Bob  
  |  
WORKS_AT  
  |  
TechCorp  
```  
  
   
  
#### 五、图模式查询（Pattern Matching）  
  
图数据库最核心能力之一是模式匹配查询。  
  
##### 查询所有用户  
  
```sql  
SELECT *  
FROM cypher('social_graph', $$  
MATCH (u:User)  
RETURN u  
$$) AS (u agtype);  
```  
  
MATCH 用于匹配图模式。  
  
例如：  
  
```  
(u:User)  
```  
  
表示：  
  
> 任意 User 节点。  
  
   
  
##### 查询好友关系  
  
```sql  
SELECT *  
FROM cypher('social_graph', $$  
MATCH (a:User)-[:FRIEND]->(b:User)  
RETURN a.name, b.name  
$$) AS (a agtype, b agtype);  
```  
  
该查询表示：  
  
```  
User -> FRIEND -> User  
```  
  
返回好友关系。  
  
   
  
#### 六、路径查询  
  
图数据库的一大优势是可以直接查询路径。  
  
##### 查询好友的好友  
  
```sql  
SELECT *  
FROM cypher('social_graph', $$  
MATCH (a:User {name:'Alice'})-[:FRIEND]->(b)-[:FRIEND]->(c)  
RETURN c  
$$) AS (c agtype);  
```  
  
该查询表示：  
  
```  
Alice -> Friend -> Friend  
```  
  
即：  
  
二度好友关系。  
  
   
  
##### 可变长度路径查询  
  
Cypher 支持可变长度路径：  
  
```sql  
MATCH (a)-[:FRIEND*1..3]->(b)  
```  
  
表示：  
  
> 查询 1 到 3 层好友关系。  
  
示例：  
  
```sql  
SELECT *  
FROM cypher('social_graph', $$  
MATCH (a:User {name:'Alice'})-[:FRIEND*1..3]->(b)  
RETURN b  
$$) AS (b agtype);  
```  
  
这种查询在社交网络分析中非常常见。  
  
   
  
#### 七、路径分析案例  
  
在推荐系统中，一个典型需求是：  
  
> 找到与用户存在社交关系的潜在联系人。  
  
示例查询：  
  
```sql  
SELECT *  
FROM cypher('social_graph', $$  
MATCH (a:User {name:'Alice'})-[:FRIEND]->()-[:FRIEND]->(candidate)  
WHERE NOT (a)-[:FRIEND]->(candidate)  
RETURN candidate  
$$) AS (candidate agtype);  
```  
  
逻辑：  
  
```  
Alice -> Friend -> Candidate  
```  
  
但排除已经是好友的用户。  
  
这就是典型的：  
  
好友推荐算法（Friend Recommendation）  
  
   
  
#### 八、SQL与图查询结合  
  
Apache AGE 的重要优势是：  
  
SQL 与图查询可以混合使用。  
  
例如，将关系表中的用户数据导入图数据库。  
  
关系表：  
  
```sql  
CREATE TABLE users (  
    id INT,  
    name TEXT  
);  
```  
  
可以通过 SQL + Cypher 导入图节点：  
  
```sql  
INSERT INTO users VALUES  
(1,'Alice'),  
(2,'Bob');  
```  
  
然后创建图节点：  
  
```sql  
SELECT *  
FROM cypher('social_graph', $$  
MATCH (n)  
RETURN n  
$$) AS (n agtype);  
```  
  
这种方式可以实现：  
  
```  
关系数据库  
      +  
图数据库  
```  
  
统一数据平台。  
  
   
  
#### 九、实际企业应用案例  
  
图数据库技术已经被广泛应用于多个行业。  
  
##### 1 社交网络  
  
社交平台需要分析：  
  
* 用户关系  
* 社群结构  
* 用户影响力  
  
图数据库可以高效执行：  
  
* 好友推荐  
* 社交传播分析  
* 社群检测  
  
   
  
##### 2 金融反欺诈  
  
银行需要识别复杂的交易网络，例如：  
  
```  
Account → Device → IP → Merchant  
```  
  
通过图分析可以发现：  
  
* 洗钱网络  
* 欺诈团伙  
* 可疑资金流  
  
   
  
##### 3 知识图谱  
  
搜索引擎和 AI 系统广泛使用知识图谱。  
  
典型结构：  
  
```  
Einstein → born_in → Germany  
Einstein → developed → Relativity  
```  
  
这种复杂关系网络非常适合图数据库管理。  
  
   
  
#### 小结  
  
通过 Apache AGE 扩展，PostgreSQL 可以直接支持图数据库能力。  
  
其核心能力包括：  
  
| 能力         | 说明                     |  
| ---------- | ---------------------- |  
| 属性图模型      | Node + Edge + Property |  
| Cypher查询语言 | 图模式匹配                  |  
| SQL融合      | SQL + Cypher           |  
| 事务支持       | 继承 PostgreSQL ACID     |  
  
图数据库特别适合处理以下问题：  
  
* 社交网络分析  
* 推荐系统  
* 金融反欺诈  
* 知识图谱  
  
通过 Apache AGE，PostgreSQL 不仅是关系数据库，还能够成为一个完整的多模数据库平台。
