#### 一、空间分析函数实践  
  
##### 1.1 空间关系判断函数  
  
空间关系判断是GIS应用的基础，PostGIS实现了OGC Simple Features规范定义的全部空间谓词。  
  
核心空间谓词函数：  
  
| 函数 | 含义 | 索引支持 | 典型应用 |  
|-----|------|---------|---------|  
| `ST_Intersects(geom1, geom2)` | 几何体相交 | 支持 GiST/SP-GiST | 区域覆盖分析 |  
| `ST_Contains(geom1, geom2)` | geom1包含geom2 | 支持 | 行政区划查询 |  
| `ST_Within(geom1, geom2)` | geom1在geom2内 | 支持 | 设备位置校验 |  
| `ST_DWithin(geom1, geom2, dist)` | 距离小于阈值 | 支持 | 邻近搜索 |  
| `ST_Touches(geom1, geom2)` | 边界接触但不重叠 | 支持 | 邻接区域分析 |  
| `ST_Crosses(geom1, geom2)` | 几何体交叉 | 支持 | 道路相交检测 |  
| `ST_Overlaps(geom1, geom2)` | 部分重叠 | 支持 | 土地利用冲突 |  
  
实践示例：商圈辐射分析  
  
```sql  
-- 场景：分析某商圈3公里内覆盖的住宅小区  
  
-- 1. 准备数据  
CREATE TABLE commercial_zones (  
    zone_id SERIAL PRIMARY KEY,  
    zone_name VARCHAR(100),  
    center_geom geometry(POINT, 4326),  
    radius_m INT  -- 辐射半径（米）  
);  
  
CREATE TABLE residential_communities (  
    community_id SERIAL PRIMARY KEY,  
    community_name VARCHAR(200),  
    boundary geometry(POLYGON, 4326),  
    household_count INT,  -- 户数  
    avg_price DECIMAL(10,2)  -- 平均房价  
);  
  
-- 2. 空间关系查询  
SELECT   
    cz.zone_name,  
    rc.community_name,  
    rc.household_count,  
    rc.avg_price,  
    ST_Distance(  
        cz.center_geom::geography,   
        ST_Centroid(rc.boundary)::geography  
    ) AS distance_m,  
    ST_Area(ST_Intersection(  
        ST_Buffer(cz.center_geom::geography, cz.radius_m)::geometry,  
        rc.boundary  
    )) / ST_Area(rc.boundary) AS coverage_ratio  -- 覆盖比例  
FROM commercial_zones cz  
JOIN residential_communities rc   
    ON ST_DWithin(  
        cz.center_geom::geography,   
        ST_Centroid(rc.boundary)::geography,   
        cz.radius_m  
    )  
WHERE cz.zone_id = 101  -- 目标商圈  
  AND rc.household_count > 500  -- 过滤小型社区  
ORDER BY coverage_ratio DESC, distance_m ASC;  
```

#### 二、空间连接与聚合分析  
  
##### 2.1 空间连接（Spatial Join）  
  
空间连接是关联不同空间数据集的核心技术，广泛用于"点落面"、"线穿面"等场景。  
  
基础空间连接示例：  
  
```sql  
-- 场景：统计各行政区的POI数量  
  
-- 方法1：ST_Intersects（精确相交）  
SELECT   
    admin.name AS district_name,  
    COUNT(poi.poi_id) AS poi_count,  
    ARRAY_AGG(DISTINCT poi.category) AS categories  
FROM administrative_districts admin  
LEFT JOIN city_poi poi   
    ON ST_Intersects(admin.boundary, poi.geom)  
GROUP BY admin.name  
ORDER BY poi_count DESC;  
  
-- 方法2：ST_Within（点完全在多边形内）  
SELECT   
    admin.name,  
    COUNT(poi.poi_id) AS poi_count  
FROM administrative_districts admin  
LEFT JOIN city_poi poi   
    ON ST_Within(poi.geom, admin.boundary)  
GROUP BY admin.name;  
```  
  
性能优化：KNN最近邻连接  
  
```sql  
-- 场景：为每个配送点分配最近的仓库  
  
-- 低效：全表笛卡尔积+排序  
SELECT   
    dp.point_id,  
    dc.center_id,  
    ST_Distance(dp.location::geography, dc.location::geography) AS dist_m  
FROM delivery_points dp  
CROSS JOIN distribution_centers dc  
ORDER BY dp.point_id, dist_m  
LIMIT 1;  -- 每个点取最近  
  
-- 高效：使用KNN操作符<->  
SELECT   
    dp.point_id,  
    (  
        SELECT dc.center_id  
        FROM distribution_centers dc  
        ORDER BY dc.location <-> dp.location  -- KNN操作符  
        LIMIT 1  
    ) AS nearest_center_id  
FROM delivery_points dp;  
  
-- 执行计划对比：  
-- 无索引：O(N×M)，10万×100 = 1000万次距离计算  
-- KNN索引：O(N×log M)，约10万次索引查找  
```  
  
多条件空间连接：  
  
```sql  
-- 场景：查找符合多重空间条件的地块  
-- 条件：在规划区内 + 距地铁<500米 + 不在生态保护区  
  
SELECT   
    lp.parcel_id,  
    lp.area_m2,  
    lp.land_use_type  
FROM land_parcels lp  
-- 条件1：在规划区内  
JOIN planning_zones pz   
    ON ST_Within(lp.geom, pz.boundary)  
    AND pz.zone_type = 'commercial'  
-- 条件2：距地铁站<500米（使用子查询+KNN）  
WHERE EXISTS (  
    SELECT 1 FROM metro_stations ms  
    WHERE ST_DWithin(lp.geom::geography, ms.location::geography, 500)  
)  
-- 条件3：不在生态保护区内  
AND NOT EXISTS (  
    SELECT 1 FROM ecological_zones ez  
    WHERE ST_Intersects(lp.geom, ez.boundary)  
)  
-- 附加属性过滤  
AND lp.area_m2 BETWEEN 1000 AND 5000  
AND lp.status = 'available';  
```  
  
##### 2.2 空间聚合分析  
  
空间分组聚合：  
  
```sql  
-- 场景：按网格统计人口密度  
  
-- 1. 创建渔网网格  
CREATE TABLE analysis_grid AS  
SELECT   
    row_id,  
    col_id,  
    ST_MakeEnvelope(  
        116.0 + col_id * 0.01,  -- 起始经度 + 列×网格宽  
        39.5 + row_id * 0.01,   -- 起始纬度 + 行×网格高  
        116.0 + (col_id+1) * 0.01,  
        39.5 + (row_id+1) * 0.01,  
        4326  
    ) AS grid_cell  
FROM generate_series(0, 49) AS row_id,  
     generate_series(0, 49) AS col_id;  
  
-- 2. 空间聚合：统计每个网格的人口  
SELECT   
    ag.row_id,  
    ag.col_id,  
    ag.grid_cell,  
    COUNT(rp.resident_id) AS population,  
    COUNT(rp.resident_id) / (ST_Area(ST_Transform(ag.grid_cell, 3857))/10000) AS density_per_km2,  
    AVG(rp.age) AS avg_age,  
    AVG(rp.income) AS avg_income  
FROM analysis_grid ag  
LEFT JOIN resident_population rp   
    ON ST_Within(rp.location, ag.grid_cell)  
GROUP BY ag.row_id, ag.col_id, ag.grid_cell  
HAVING COUNT(rp.resident_id) > 0;  -- 过滤空网格  
```  
  
空间窗口聚合：  
  
```sql  
-- 场景：计算每个POI周边500米内的竞争店铺数量  
  
SELECT   
    p.poi_id,  
    p.name,  
    p.category,  
    COUNT(DISTINCT c.poi_id) AS competitor_count,  
    AVG(c.rating) AS avg_competitor_rating,  
    STRING_AGG(DISTINCT c.name, ', ' ORDER BY c.name) AS competitor_names  
FROM city_poi p  
LEFT JOIN city_poi c   
    ON p.category = c.category  -- 同类竞争  
    AND p.poi_id != c.poi_id    -- 排除自身  
    AND ST_DWithin(  
        p.geom::geography,   
        c.geom::geography,   
        500  -- 500米竞争半径  
    )  
GROUP BY p.poi_id, p.name, p.category  
ORDER BY competitor_count DESC  
LIMIT 20;  
```  
  
时空聚合分析：  
  
```sql  
-- 场景：分析共享单车使用热点的时空分布  
  
SELECT   
    DATE_TRUNC('hour', trip.start_time) AS time_bucket,  
    ST_SnapToGrid(trip.start_location, 0.001) AS location_grid,  -- ~100米网格  
    COUNT(*) AS trip_count,  
    AVG(EXTRACT(EPOCH FROM (trip.end_time - trip.start_time))) AS avg_duration_sec,  
    AVG(trip.distance_m) AS avg_distance_m  
FROM bike_trips trip  
WHERE trip.start_time >= '2025-01-01'  
  AND trip.start_time < '2025-02-01'  
GROUP BY time_bucket, location_grid  
HAVING COUNT(*) >= 10  -- 过滤低频网格  
ORDER BY trip_count DESC  
LIMIT 100;  
```

方案二：集成外部地理编码API  
  
```python  
# Python应用层集成高德/百度/Google地理编码  
import requests  
import psycopg  
  
def geocode_address(address: str, api_key: str) -> tuple[float, float] | None:  
    """调用高德地理编码API"""  
    url = "https://restapi.amap.com/v3/geocode/geo"  
    params = {  
        "key": api_key,  
        "address": address,  
        "city": "beijing",  
        "output": "json"  
    }  
      
    response = requests.get(url, params=params)  
    data = response.json()  
      
    if data.get("status") == "1" and data.get("geocodes"):  
        location = data["geocodes"][0]["location"]  # "116.481485,39.990464"  
        lon, lat = map(float, location.split(","))  
        return lon, lat  
    return None  
  
# 批量地理编码并入库  
def batch_geocode_and_store(addresses: list[dict], conn: psycopg.Connection):  
    """批量处理地址地理编码"""  
    for item in addresses:  
        coords = geocode_address(item["address"], API_KEY)  
        if coords:  
            lon, lat = coords  
            with conn.cursor() as cur:  
                cur.execute("""  
                    INSERT INTO poi_locations (name, address, geom, source)  
                    VALUES (%s, %s, ST_SetSRID(ST_MakePoint(%s, %s), 4326), %s)  
                    ON CONFLICT (name, address) DO UPDATE  
                    SET geom = EXCLUDED.geom, updated_at = NOW()  
                """, (item["name"], item["address"], lon, lat, "amap"))  
            conn.commit()  
```  
  
方案三：预构建地址索引表  
  
```sql  
-- 针对高频地址预计算坐标，提升查询性能  
  
CREATE TABLE address_index (  
    address_hash BIGINT PRIMARY KEY,  -- MD5(address)的BIGINT  
    address_text TEXT,  
    geom geometry(POINT, 4326),  
    confidence FLOAT,  -- 匹配置信度  
    source VARCHAR(20),  -- 数据来源  
    created_at TIMESTAMPTZ DEFAULT NOW()  
);  
  
-- 创建地址前缀索引（支持模糊匹配）  
CREATE INDEX idx_address_prefix ON address_index   
USING GIN (address_text gin_trgm_ops);  
  
-- 地理编码查询函数  
CREATE OR REPLACE FUNCTION fast_geocode(query_address TEXT)  
RETURNS TABLE (  
    matched_address TEXT,  
    geom geometry(POINT, 4326),  
    confidence FLOAT,  
    similarity FLOAT  
) AS $$  
BEGIN  
    RETURN QUERY  
    SELECT   
        ai.address_text,  
        ai.geom,  
        ai.confidence,  
        similarity(ai.address_text, query_address) AS sim  
    FROM address_index ai  
    WHERE ai.address_text % query_address  -- trgm模糊匹配  
      AND similarity(ai.address_text, query_address) > 0.6  
    ORDER BY sim DESC, ai.confidence DESC  
    LIMIT 5;  
END;  
$$ LANGUAGE plpgsql;  
```  
  
##### 3.2 逆地理编码（坐标→地址）  
  
方案一：空间连接行政区划表  
  
```sql  
-- 准备行政区划表（含地址模板）  
CREATE TABLE administrative_divisions (  
    div_id SERIAL PRIMARY KEY,  
    div_name VARCHAR(100),  
    div_level VARCHAR(20),  -- province/city/district/street  
    boundary geometry(MULTIPOLYGON, 4326),  
    address_template VARCHAR(200)  -- 如'{province}{city}{district}'  
);  
  
-- 逆地理编码函数  
CREATE OR REPLACE FUNCTION reverse_geocode(  
    query_point geometry(POINT, 4326),  
    detail_level VARCHAR(20) DEFAULT 'district'  
)  
RETURNS TABLE (  
    div_name VARCHAR,  
    div_level VARCHAR,  
    full_address TEXT  
) AS $$  
BEGIN  
    RETURN QUERY  
    SELECT   
        ad.div_name,  
        ad.div_level,  
        REPLACE(  
            REPLACE(  
                REPLACE(ad.address_template, '{province}',   
                    (SELECT div_name FROM administrative_divisions WHERE div_level='province' AND ST_Contains(boundary, query_point) LIMIT 1)  
                ),  
                '{city}',   
                (SELECT div_name FROM administrative_divisions WHERE div_level='city' AND ST_Contains(boundary, query_point) LIMIT 1)  
            ),  
            '{district}',  
            (SELECT div_name FROM administrative_divisions WHERE div_level='district' AND ST_Contains(boundary, query_point) LIMIT 1)  
        ) AS full_address  
    FROM administrative_divisions ad  
    WHERE ST_Contains(ad.boundary, query_point)  
      AND ad.div_level <= detail_level  -- 控制返回粒度  
    ORDER BY   
        CASE ad.div_level   
            WHEN 'province' THEN 1   
            WHEN 'city' THEN 2   
            WHEN 'district' THEN 3   
            ELSE 4   
        END;  
END;  
$$ LANGUAGE plpgsql;  
  
-- 调用示例  
SELECT * FROM reverse_geocode(  
    ST_SetSRID(ST_MakePoint(116.397, 39.908), 4326),  
    'district'  
);  
-- 输出：北京市 | city | 北京市  
--       东城区 | district | 北京市东城区  
```

```sql  
        ak.address_text,  
        ST_Distance(query_point::geography, ak.geom::geography),  
        0.9 + ak.popularity * 0.001 AS confidence  
    FROM address_keywords ak  
    WHERE ST_DWithin(query_point::geography, ak.geom::geography, radius_m)  
      
    UNION ALL  
      
    -- 次选行政区划  
    SELECT   
        'admin'::VARCHAR,  
        ad.div_name,  
        NULL,  
        ST_Distance(query_point::geography, ST_Centroid(ad.boundary)::geography),  
        0.7 AS confidence  
    FROM administrative_divisions ad  
    WHERE ST_DWithin(query_point::geography, ST_Centroid(ad.boundary)::geography, radius_m * 2)  
      AND ad.div_level IN ('district', 'street')  
      
    ORDER BY confidence DESC, distance_m ASC  
    LIMIT 3;  
END;  
$$ LANGUAGE plpgsql;
```  

```sql  
-- 场景：配送车从仓库出发，依次访问5个配送点  
  
-- 1. 计算所有点对之间的最短路径成本  
CREATE TEMP TABLE distance_matrix AS  
SELECT   
    start_node,  
    end_node,  
    agg_cost AS travel_seconds  
FROM pgr_dijkstraCost(  
    'SELECT edge_id AS id, source_node, target_node, cost_seconds FROM road_network',  
    ARRAY[1001, 1102, 1205, 1308, 1411, 1514],  -- 仓库+5个配送点  
    ARRAY[1001, 1102, 1205, 1308, 1411, 1514],  
    directed := true  
);  
  
-- 2. 应用层使用动态规划/遗传算法求解TSP  
-- （此处简化：按输入顺序访问）  
WITH delivery_order AS (  
    SELECT unnest(ARRAY[1001, 1102, 1205, 1308, 1411, 1514]) AS node_id,  
           generate_series(1, 6) AS visit_order  
)  
SELECT   
    do1.visit_order,  
    do1.node_id AS from_node,  
    do2.node_id AS to_node,  
    dm.travel_seconds,  
    route.geom AS path_geom  
FROM delivery_order do1  
JOIN delivery_order do2 ON do1.visit_order + 1 = do2.visit_order  
JOIN distance_matrix dm ON dm.start_node = do1.node_id AND dm.end_node = do2.node_id  
LEFT JOIN LATERAL (  
    SELECT ST_Collect(geom) AS geom  
    FROM pgr_dijkstra(  
        'SELECT edge_id AS id, source_node, target_node, geom FROM road_network',  
        do1.node_id, do2.node_id, false  
    ) AS r  
    JOIN road_network rn ON r.edge = rn.edge_id  
) route ON true  
ORDER BY do1.visit_order;  
```

```bash  
import subprocess  
import pathlib  
  
def import_shapefiles(shapefile_dir: str, target_schema: str, srid: int = 4326):  
    """批量导入Shapefile到PostGIS"""  
    for shp_file in pathlib.Path(shapefile_dir).glob("*.shp"):  
        table_name = shp_file.stem.lower()  
          
        cmd = [  
            "shp2pgsql",  
            "-s", str(srid),  
            "-I",  # 创建索引  
            "-D",  # dump格式  
            "-W", "UTF-8",  
            str(shp_file),  
            f"{target_schema}.{table_name}"  
        ]  
          
        # 执行导入  
        result = subprocess.run(  
            cmd,  
            stdout=subprocess.PIPE,  
            stderr=subprocess.PIPE,  
            text=True,  
            env={"PGPASSWORD": "your_password"}  
        )  
          
        if result.returncode == 0:  
            print(f"✓ Imported {table_name}")  
        else:  
            print(f"✗ Failed {table_name}: {result.stderr}")
```  

```sql  
-- 识别早高峰拥堵路段（速度<20km/h且流量>阈值）  
WITH peak_hours AS (  
    SELECT segment_id, timestamp, avg_speed_kmh, vehicle_count  
    FROM traffic_flow  
    WHERE timestamp >= CURRENT_DATE + INTERVAL '7 hours'  -- 7:00  
      AND timestamp < CURRENT_DATE + INTERVAL '9 hours'    -- 9:00  
),  
congested_segments AS (  
    SELECT   
        segment_id,  
        AVG(avg_speed_kmh) AS avg_speed,  
        AVG(vehicle_count) AS avg_volume,  
        COUNT(*) AS sample_count  
    FROM peak_hours  
    GROUP BY segment_id  
    HAVING AVG(avg_speed_kmh) < 20   
       AND AVG(vehicle_count) > 100  
       AND COUNT(*) >= 10  -- 至少10个采样点  
)  
SELECT   
    rs.road_name,  
    rs.road_class,  
    cs.avg_speed,  
    cs.avg_volume,  
    ST_Length(ST_Transform(rs.geom, 3857)) AS segment_length_m,  
    -- 计算拥堵影响范围（缓冲+相交）  
    ST_Buffer(rs.geom::geography, 200)::geometry AS influence_zone  
FROM congested_segments cs  
JOIN road_segments rs ON cs.segment_id = rs.segment_id  
ORDER BY cs.avg_speed ASC, cs.avg_volume DESC  
LIMIT 50;
```  

```sql  
-- 动态调整路网成本：事件影响区域内通行成本×3  
CREATE OR REPLACE FUNCTION get_dynamic_cost(  
    edge_id BIGINT,   
    base_cost FLOAT,  
    event_time TIMESTAMPTZ  
)  
RETURNS FLOAT AS $$  
DECLARE  
    affected BOOLEAN;  
BEGIN  
    SELECT EXISTS(  
        SELECT 1 FROM traffic_events te  
        WHERE te.start_time <= event_time  
          AND te.end_time >= event_time  
          AND ST_DWithin(  
                (SELECT geom FROM road_segments WHERE segment_id = edge_id)::geography,  
                te.geom::geography,  
                te.affected_radius_m  
          )  
    ) INTO affected;  
      
    RETURN CASE WHEN affected THEN base_cost * 3 ELSE base_cost END;  
END;  
$$ LANGUAGE plpgsql;  
  
-- 使用动态成本计算路径  
SELECT * FROM pgr_dijkstra(  
    $$  
    SELECT   
        segment_id AS id,  
        source_node,  
        target_node,  
        get_dynamic_cost(segment_id, cost_seconds, NOW()) AS cost,  
        get_dynamic_cost(segment_id, reverse_cost_seconds, NOW()) AS reverse_cost  
    FROM road_segments  
    $$,  
    1001, 2050,  
    directed := true  
);
```  

```sql  
-- 检测2020-2025年林地转为建设用地的区域  
  
WITH land_use_2020 AS (  
    SELECT parcel_id, parcel_geom, land_use_code  
    FROM land_use_classification  
    WHERE survey_year = 2020  
),  
land_use_2025 AS (  
    SELECT parcel_id, parcel_geom, land_use_code  
    FROM land_use_classification  
    WHERE survey_year = 2025  
),  
changed_parcels AS (  
    SELECT   
        p2020.parcel_id,  
        p2020.parcel_geom AS geom_2020,  
        p2020.land_use_code AS code_2020,  
        p2025.land_use_code AS code_2025,  
        ST_Area(ST_Transform(p2020.parcel_geom, 4490)) / 10000 AS area_ha  
    FROM land_use_2020 p2020  
    JOIN land_use_2025 p2025   
        ON ST_Equals(p2020.parcel_geom, p2025.parcel_geom)  -- 同一地块  
    WHERE p2020.land_use_code LIKE '03%'  -- 2020年为林地  
      AND p2025.land_use_code LIKE '20%'  -- 2025年为建设用地  
)  
SELECT   
    cp.parcel_id,  
    cp.area_ha,  
    -- 叠加生态红线，判断是否违规  
    CASE   
        WHEN EXISTS(  
            SELECT 1 FROM ecological_redlines er  
            WHERE ST_Intersects(cp.geom_2020, er.boundary)  
              AND er.protection_level IN ('core', 'buffer')  
        ) THEN 'VIOLATION'  
        ELSE 'LEGAL'  
    END AS compliance_status,  
    -- 关联遥感影像验证  
    (  
        SELECT si.satellite_name || '_' || TO_CHAR(si.acquisition_time, 'YYYYMMDD')  
        FROM satellite_imagery si  
        WHERE ST_Intersects(si.footprint, cp.geom_2020)  
          AND si.acquisition_time BETWEEN '2025-01-01' AND '2025-12-31'  
          AND si.cloud_cover_pct < 10  
        ORDER BY si.acquisition_time DESC  
        LIMIT 1  
    ) AS verification_image  
FROM changed_parcels cp  
ORDER BY   
    compliance_status DESC,  -- 违规优先  
    cp.area_ha DESC;
```  

生态敏感性评估：  
  
```sql  
-- 综合多因子评估区域生态敏感性  
CREATE OR REPLACE FUNCTION calculate_ecological_sensitivity(  
    query_geom geometry,  
    weights JSONB DEFAULT '{"slope":0.3,"vegetation":0.25,"water":0.2,"redline":0.25}'  
)  
RETURNS TABLE (  
    sensitivity_score FLOAT,  -- 0-100  
    sensitivity_level VARCHAR(10),  -- low/medium/high/critical  
    contributing_factors JSONB  
) AS $$  
DECLARE  
    w JSONB := weights;  
    slope_score FLOAT;  
    veg_score FLOAT;  
    water_score FLOAT;  
    redline_score FLOAT;  
BEGIN  
    -- 因子1：坡度（从DEM计算）  
    SELECT AVG(ST_Value(rast, 1, query_geom)) INTO slope_score  
    FROM terrain_slope_raster  
    WHERE ST_Intersects(rast, query_geom);  
      
    -- 因子2：植被覆盖（NDVI）  
    SELECT AVG(ST_Value(rast, 4, query_geom)) INTO veg_score  -- Band4=NDVI  
    FROM satellite_ndvi  
    WHERE ST_Intersects(rast, query_geom)  
      AND acquisition_time >= CURRENT_DATE - INTERVAL '1 year';  
      
    -- 因子3：距水体距离  
    SELECT MIN(ST_Distance(query_geom::geography, w.geom::geography)) / 1000   
    INTO water_score  
    FROM water_bodies w;  
    water_score := GREATEST(0, 10 - water_score) / 10;  -- 归一化  
      
    -- 因子4：生态红线  
    SELECT CASE   
        WHEN EXISTS(SELECT 1 FROM ecological_redlines   
                   WHERE ST_Contains(boundary, query_geom) AND protection_level='core') THEN 1.0  
        WHEN EXISTS(SELECT 1 FROM ecological_redlines   
                   WHERE ST_Intersects(boundary, query_geom)) THEN 0.7  
        ELSE 0.0  
    END INTO redline_score;  
      
    -- 加权融合  
    sensitivity_score :=   
        (slope_score * (w->>'slope')::FLOAT) +  
        (veg_score * (w->>'vegetation')::FLOAT) +  
        (water_score * (w->>'water')::FLOAT) +  
        (redline_score * (w->>'redline')::FLOAT);  
      
    sensitivity_score := sensitivity_score * 100;  -- 转0-100分  
      
    -- 分级  
    sensitivity_level := CASE   
        WHEN sensitivity_score >= 75 THEN 'critical'  
        WHEN sensitivity_score >= 50 THEN 'high'  
        WHEN sensitivity_score >= 25 THEN 'medium'  
        ELSE 'low'  
    END;  
      
    RETURN QUERY SELECT   
        sensitivity_score,  
        sensitivity_level,  
        jsonb_build_object(  
            'slope', ROUND(slope_score::NUMERIC, 2),  
            'vegetation', ROUND(veg_score::NUMERIC, 2),  
            'water', ROUND(water_score::NUMERIC, 2),  
            'redline', ROUND(redline_score::NUMERIC, 2)  
        );  
END;  
$$ LANGUAGE plpgsql;
```  

#### 七、最佳实践与常见问题  
  
##### 7.1 空间查询优化清单  
  
```  
支持 索引使用  
   • 确保空间列有GiST/SP-GiST/BRIN索引  
   • 使用ST_DWithin而非ST_Distance+比较  
   • 显式使用&&边界框预过滤  
  
支持 坐标系统规范  
   • 存储统一用SRID 4326  
   • 计算距离用::geography或转换到投影坐标  
   • Web展示转3857  
  
支持 几何简化  
   • 渲染前使用ST_Simplify降低复杂度  
   • 大面数据先用ST_Centroid快速过滤  
  
支持 查询重写  
   • 避免在空间列上使用函数（除非有函数索引）  
   • 使用CTE或子查询分解复杂逻辑  
   • 合理设置LIMIT减少返回数据量  
  
支持 统计信息  
   • 定期ANALYZE更新空间列统计  
   • 对高基数空间列增加STATISTICS目标  
```  
  
##### 7.2 常见错误与解决方案  
  
| 错误现象 | 可能原因 | 解决方案 |  
|---------|---------|---------|  
| 查询慢且未用索引 | 未创建空间索引或查询未触发索引 | `CREATE INDEX ... USING GIST`; 使用`ST_DWithin` |  
| 距离计算结果异常 | 混用geometry和geography单位 | 明确转换：`geom::geography` |  
| ST_Intersects返回过多结果 | 边界框过滤后未精确计算 | 确保查询包含精确谓词 |  
| 导入后查询报错 | SRID不一致 | 统一使用`ST_SetSRID(geom, 4326)` |  
| 多边形显示异常 | 环方向错误或自相交 | 使用`ST_MakeValid`或`ST_ForceRHR` |  
| 内存溢出 | 大几何未简化 | 查询前`ST_Simplify(geom, tolerance)` |  
  
##### 7.3 生产环境配置建议  
  
```sql  
-- postgresql.conf 空间查询优化参数  
shared_buffers = 8GB                    # 系统内存25%  
effective_cache_size = 24GB             # 系统内存75%  
work_mem = 256MB                        # 复杂查询排序/哈希  
maintenance_work_mem = 2GB              # 索引构建  
random_page_cost = 1.1                  # SSD存储  
geqo_threshold = 15                     # 多表连接使用遗传算法阈值  
  
-- PostGIS特定配置  
# postgis.conf (可单独加载)  
postgis.gdal_enabled_drivers = 'GTiff,JP2OpenJPEG'  # 启用栅格驱动  
postgis.enable_outdb_rasters = true     # 支持外部栅格  
  
-- 监控视图  
CREATE VIEW spatial_query_stats AS  
SELECT   
    query,  
    calls,  
    total_time,  
    mean_time,  
    rows,  
    shared_blks_hit,  
    shared_blks_read  
FROM pg_stat_statements  
WHERE query ~* 'ST_(Intersects|Within|Distance|Buffer)'  
ORDER BY mean_time DESC  
LIMIT 20;  
```
